# Runbooks

Step-by-step operational procedures for the WebRTC speaker streaming project.

## When to Create a Runbook

- Deploy or release procedures
- Debugging common failures (ICE failures, TTS model loading, etc.)
- Environment setup for new machines
- Recovery procedures for known failure modes

## Runbook Template

```markdown
# Runbook: [Title]

## Purpose
What does this procedure accomplish?

## Prerequisites
- [ ] Required tools installed
- [ ] Required access/credentials

## Steps
1. Step one
2. Step two
3. Step three

## Verification
How to confirm the procedure worked.

## Rollback
How to undo if something goes wrong.

## Related
- Session: S-YYYY-MM-DD-HHMM-slug
- ADR: ADR-XXXX
```
